AECCT_HLS v12｜M0～M24 搭建規劃（取代 v11.7 版；單一 X_WORK 修訂）

==================================================
全域規則（所有里程碑共用）
==================================================

A. Contract / 邊界 / 風格
- Top 外部介面固定不變：
  - ctrl_cmd  : ac_channel<ac_int<16,false>>
  - ctrl_rsp  : ac_channel<ac_int<16,false>>
  - data_in   : ac_channel<ac_int<32,false>>
  - data_out  : ac_channel<ac_int<32,false>>
- DESIGN 嚴禁 include trace/golden/weights.h；權重只能由 TB 透過 LOAD_W/PARAM_RX 串流寫入 SRAM。
- DESIGN 嚴守 HLS-safe subset：優先 ac_int / ac_fixed / ac_ieee_float<binary32>，禁止 host-only 技巧。
- 所有 shape 以 ModelDesc.h / ModelShapes.h 為 SSOT。
- token 是列(Row)，d 是行(Colume)；線性化以 d/行(Colume) 為 inner-most contiguous。

B. 量化總原則（v12.1 revB）
- v1 先只實作 AECCT 原生 linear quant：ternary weights + INT8 activations。
- 其他路徑先維持 FP32；未來若要導入 E4M3，必須以獨立條款逐步導入。
- s_x 先採固定值；量化時必須做 clamp / sat，禁止 overflow / wrap-around。
- 權重端正式保留 s_w 語意，但主流程只存 inv_s_w。
- grouping / zone / SET_QCFG / QuantDesc 正式配置在本版只保留 reserved 身分，不進主 datapath。

C. 每個 milestone 都必須附帶
- docs/milestones/Mxx_report.md
- docs/milestones/Mxx_artifacts/
  - file_manifest.txt
  - diff.patch
  - build.log
  - run_tb.log
  - verdict.txt
- 必跑 gates：
  - check_design_purity
  - check_interface_lock
  - check_repo_hygiene
  - check_tb_flow（Top/bring-up 類 TB）
